# 旧実装の評価フローメモ

正本実装は `master_thesis_modules/scripts/master_v5.py` です。入力は `dict[str, pandas.DataFrame]` で、各DataFrameは `timestamp` と既存ノード番号列を持ちます。

評価順序は次の通りです。

1. `50000000`, `50000010` から、属性リスク `40000000`, `40000001` を作る。
2. `50000100`-`50000103` と任意の `60010002` から、動作リスク `40000010`-`40000016` を作る。
3. 物体座標と本人座標から、周辺物体リスク `40000100`-`40000102` を作る。
4. スタッフ座標・速度と本人座標から、スタッフ関連リスク `40000110`, `40000111` を作る。
5. 属性リスクから `30000000` を作る。
6. 動作リスクから、AHP重みで `30000001` を作る。
7. 周辺物体リスクから、AHP重みで `30000010` を作る。
8. スタッフ関連リスクから、Fuzzy推論で `30000011` を作る。
9. `30000000`, `30000001` から、重み `[0.1, 0.9]` で `20000000` を作る。
10. `30000010`, `30000011` から、Fuzzy推論で `20000001` を作る。
11. `20000000`, `20000001` から、Fuzzy推論で `10000000` を作る。

新実装では `master_v5.py` を保持したまま、`risk_core` の各モジュールでこの流れを再現します。AHPとFuzzyは、既定の軽量実装に加えて、既存CSVを読むためのadapterを用意しています。
